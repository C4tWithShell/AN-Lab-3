Configuring the connection between OVSs and SDN controller

I started with the following topology (Figure 2).


Figure 2 - Initial topology

Let’s start with OpenVSwitch (OVS). For each of them, I have set a static IP address for the eth0 interface (management interface). Figure 3 shows the example.


Figure 3 - Configuration of the eth0 interface

So here are the IP addresses for OVSs:
1 - 172.12.1.1
2 - 172.12.1.2
3 - 172.12.1.3

Then I ran them and configured the br0 interface as shown below:

ovs-vsctl set bridge br0 other-config:datapath-id=0000000000000001 \
            -- set bridge br0 fail_mode=secure \
            -- set-controller br0 tcp:172.12.1.254:6653 \
            -- set controller br0 connection-mode=out-of-band \
            -- --if-exists del-port eth0

For the second switch id = 0000000000000002 and the third - 0000000000000003. This value will be used in the SDN controller.

This command gives a new id for the bridge br0 and puts br0 into the so-called “fail-secure” mode. This simply means that the OpenFlow flow table starts empty. Then I set the controller for this interface and set the connection mode = out-of-band.

Import

The management interface eth0 should not be a part of the software bridge br0. That’s why I’m deleting it from it.

Now let’s make it work with VLANs.

I’m going to create two VLANs - 100 and 200. First I changed my network by adding PCs and connecting OVSs as shown in Figure 6.


Figure 6 - Network topology

Now let’s move on to the faucet. I need to edit the Faucet network configuration to specify the desired datapath (OVS). I must specify each physical switch in the virtual datapath, as well as each connected OpenFlow port.

I started by creating two VLANs: hr with vid 100 and managment - 200. For each of them, I have configured an IP address for the faucet routing interface (faucet_vips). Then I enabled inter-VLAN routing between these two VLANs. In the faucet, you do this by creating a router and specifying which VLANs can route between each other (Figure 7).


Figure 7 - The first part of the faucet configuration

Then I started setting up OpenVSwitches. I have set native_vlan for ports with hosts - a VLAN associated with untagged packets coming in and out of this interface. The ports that connect OVSs have been configured as a stack (a configuration block for interface-level stacking configuration) that requires a port and dp - the name or dp_id of the dp connected to this port (Figures 8, 9, 10).


Figure 8 - The second part of the faucet configuration


Figure 9 - The third part of the faucet configuration


Figure 10 - The fouth part of the faucet configuration

The check_faucet_config command can be used to verify faucet has correctly interpreted your configuration before loading it. This script will either return an error, or in the case of successfully parsing the configuration it will return a JSON object containing the entire faucet configuration that would be loaded (including any default settings). So in my case it’s alright.

After that, I restarted the faucet using the command pkill -HUP ryu-manager.

Now let’s analyze the flow. To do this, I dumped the flows from OVS1 into files and then compared them.

ovs-ofctl dump-flows br0 | cut -d , -f 3,7- > flows_without_routing

ovs-ofctl dump-flows br1 | cut -d , -f 3,7- > flows_with_routing

Figure 14 shows the result of the diff flows_without_routing flows_with_routing | grep table=1.


Figure 14 - The first table

We look at the added streams for handling ARP packets and packets directed to the MAC router (0e:00:00:00:00:01)
Enabling cross-VLAN routing adds the streams that handle arp packets across the different VLANs.

Figure 15 shows the result of the diff flows_without_routing flows_with_routing | grep table=2.


Figure 15 - The second table

This table contains the forwarding information for IP packets.

Figure 16 shows the result of the diff flows_without_routing flows_with_routing | grep table=3.


Figure 16 - The third table

This table sends the ARP packets to the controller.

Figure 17 - The fourth table
